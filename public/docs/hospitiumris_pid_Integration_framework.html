Technical Framework

HospitiumRIS PID Integration Framework

A practical architecture showing where each PID provider sits, what data flows through them, and how hospitals use the outputs for research visibility, compliance, and reporting.

System Architecture

Below is the technological framework for integrating ORCID, Crossref, and DataCite within HospitiumRIS.

Hospital / University Teaching Hospital │ ▼ HospitiumRIS Core Platform Researcher Profiles │ Projects │ Publications │ Datasets │ Grants │ Ethics │ Clinical Studies │ ├── ORCID Integration │ ├─ Researcher authentication │ ├─ ORCID iD capture │ ├─ Researcher profile synchronization │ └─ Push verified affiliations, works, funding, peer review │ ├── Crossref Integration │ ├─ Retrieve publication DOI metadata │ ├─ Validate journal articles, conference papers, books │ ├─ Link authors, affiliations, references, funders │ └─ Support citation and publication reporting │ └── DataCite Integration ├─ Register or retrieve dataset DOIs ├─ Link datasets to projects, grants, ethics approvals ├─ Capture related identifiers and provenance └─ Support FAIR data and Open Science reporting

Role of Each PID Provider

PID System Function inside HospitiumRIS Main Value to Hospitals
ORCID Identifies researchers, clinicians, students, and collaborators Solves researcher name ambiguity and improves researcher visibility
Crossref Connects publications with DOI metadata Tracks journal articles, citations, funders, affiliations, and research outputs
DataCite Connects datasets, software, instruments, reports, protocols, and clinical research objects Supports FAIR data, dataset citation, and research reproducibility
ORCID supports API-based connections between systems and researcher records, including affiliations, outputs, funding, peer review, and research resources. Crossref provides DOI metadata retrieval through its REST API, including publication, citation, relationship, funder, and licence metadata. DataCite's REST API supports retrieval, creation, and update of DOI metadata, especially for datasets and research objects.

How the Data Flow Works

Researcher logs into HospitiumRIS
↓
Connects ORCID iD
↓
HospitiumRIS imports researcher profile and verified works
↓
Publications are checked against Crossref DOI metadata
↓
Datasets and research objects are checked or registered through DataCite
↓
HospitiumRIS creates a unified research record
↓
Hospital leadership gets dashboards and reports

Strategic Value for HospitiumRIS

HospitiumRIS becomes the hospital's PID-enabled Research Information Management System, helping hospitals:

Improve researcher visibility
Reduce duplicate researcher records
Track publications and datasets
Link grants to outputs
Support Open Science and FAIR data
Strengthen institutional reporting
Prepare for funder and ministry reporting
Make hospital research discoverable globally
In short: ORCID identifies the people, Crossref identifies the publications, and DataCite identifies the datasets and research objects. HospitiumRIS brings them together into one institutional research intelligence system.
